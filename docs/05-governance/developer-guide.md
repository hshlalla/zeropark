# 개발 문서 작성 규칙

앞으로 모든 개발 프로젝트는 동일한 버전관리 규칙을 적용합니다.

## 필수 규칙

1. 모든 프로젝트는 반드시 `docs` 폴더를 가진다.
2. `docs/project.yml`, `docs/01-planning/development-spec.md`, `docs/02-progress/progress.md`는 필수 작성한다.
3. 모든 작업은 Task ID를 가진다. 예: `DEV-001`
4. 커밋 메시지에는 반드시 Task ID를 포함한다.
5. 브랜치명에도 Task ID를 포함한다.
6. 진행 상황은 `docs/02-progress/progress.md`에 업데이트한다.
7. 막힌 부분은 숨기지 말고 `블로커` 항목에 작성한다.
8. 완료 기준은 코드 작성이 아니라 테스트, 리뷰, 문서 업데이트까지 포함한다.
9. AI 코딩 도구를 사용할 경우 `AGENTS.md`의 규칙을 따르게 한다.
10. `main` 브랜치 직접 push는 금지한다.

## 팀 공유 문구

```md
앞으로 모든 개발 프로젝트는 동일한 버전관리 규칙을 적용합니다.

프로젝트 시작 시 기본 템플릿으로 `docs/`, `.gitlab/`, `AGENTS.md`를 반드시 포함합니다.

개발자는 `docs` 문서를 기준으로 작업하고, AI 코딩 도구나 LLM을 사용할 경우 반드시 `AGENTS.md`의 규칙을 따르게 합니다.

커밋, 브랜치, MR에는 반드시 Task ID를 포함해야 하며, main 브랜치 직접 커밋은 금지합니다.

이 규칙은 개발자를 통제하기 위한 목적이 아니라, 프로젝트 진행 상황을 추적하고, 작업 누락과 장애 요인을 빠르게 발견하기 위한 기준입니다.
```
