# 3-Tier 역할 기반 유저 테스트(검증) 계획 — 설계

- 작성일: 2026-06-17
- 대상: Zeropark (Gateway `:8080` + Control Plane `:8090` + Web)
- 목적: 솔루션을 공급사(삼성/LG 등)에 출하하기 전, **공급 체인 3역할 관점**에서
  전 기능이 올바르게 동작하고 **권한 경계가 새지 않는지** 검증한다.

## 1. 역할 모델

| 역할 | 정체 | 진입점 | 인증 |
|---|---|---|---|
| 슈퍼관리자 | 우리 회사 | Control Plane `:8090` | `X-Admin-Token` 헤더 |
| 관리자 | 솔루션을 공급받는 회사 | Gateway `:8080` | admin role JWT |
| 유저 | 공급사의 실제 사용자 | Gateway `:8080` | user role JWT |

역할 시뮬레이션: dev에서는 guest 로그인이 admin/user JWT를 즉석 발급한다
(`POST /api/v1/auth/guest/login?role=admin|user`). 슈퍼관리자는 Control Plane의
`ZEROPARK_CP_ADMIN_TOKEN`. 실계정 흐름(로그인→토큰)도 별도 1건 수동 확인한다.

## 2. 방식 / 환경

- **두 종류 산출물**: 자동화 시나리오 스크립트 + 수동 UAT 체크리스트.
- **현재 가용 환경 기준**: 시작 시 `/health`·`/api/v1/profile`로 가용 capability를
  읽어, 외부 의존 미충족 기능은 **SKIP(사유 명시)** 으로 처리하고 불합격으로 보지 않는다.
  - 검색 API 키 없음 → `research` SKIP
  - Docker 없음 → `super_agent`의 python 도구 SKIP (LLM 루프 자체는 검증)
  - 이미지/TTS는 실 OpenAI 호출(비용 소액) — 현재 키로 검증 대상에 포함
- 합격/불합격은 **PASS / FAIL / SKIP** 3분류로 집계한다.

## 3. Part 1 — 공급 체인 E2E (단일 줄기, 자동화)

발급 → 셋업 → 사용 → 권한격리 → 회수를 한 줄기로 검증한다.

1. **슈퍼관리자**: 배포본 등록 → 라이선스 키 발급 확인.
2. 게이트웨이가 그 라이선스로 하트비트 → Control Plane에서 **online** 전환 확인.
3. **슈퍼관리자**: 프로파일에 `features.image=false` 설정 → 다음 하트비트 후
   게이트웨이 `/api/v1/profile`에서 `image`가 사라짐(**hot-reload**) 확인.
4. **관리자**: admin-전용 RAG 컬렉션 생성 + 문서 업로드 → 그 컬렉션에 고정된
   "인사봇"(chat+RAG mode) 에이전트를 발행(published).
5. **유저**: 발행된 인사봇이 목록에 보이고, 질문 시 업로드 문서 기반으로 답변.
6. **권한 경계(핵심)**: 유저가
   - (a) admin 엔드포인트 호출 → 403,
   - (b) admin-전용 컬렉션 문서가 검색 결과에서 제외,
   - (c) Control Plane 접근 불가.
7. **관리자**가 그 유저를 role 강등 → 유저의 **기존 토큰 즉시 무효화(401)** 확인.
8. **슈퍼관리자**: 라이선스 중지 → 게이트웨이 하트비트 **403** 확인.

## 4. Part 2 — 역할별 기능 매트릭스 (자동 = 스크립트, 수동 = 체크리스트)

### 4.1 슈퍼관리자 (Control Plane)
| 기능 | 방식 | 합격 기준 |
|---|---|---|
| 배포본 CRUD | 자동 | 생성/조회/수정/삭제 200, 라이선스 키 발급 |
| online/offline 감시 | 자동 | 하트비트 후 online, grace 초과 후 offline |
| 라이선스 활성/중지 | 자동 | 중지 시 하트비트 403 |
| 프로파일 hot-reload | 자동 | features/branding 변경이 게이트웨이에 반영 |
| 사용량 시계열 | 자동 | 하트비트마다 usage-history 행 누적 |
| offline 웹훅 알림 | 수동 | 웹훅 설정 후 배포본 중단 시 1회 발사 |
| 대시보드 UI | 수동 | `/`에서 목록·상태·프로파일 편집 동작 |

### 4.2 관리자 (Gateway admin)
| 기능 | 방식 | 합격 기준 |
|---|---|---|
| 에이전트 생성/수정/삭제/발행 | 자동 | CRUD 200, published만 유저 노출 |
| 에이전트 파라미터(model/temp/prompt/변수/지식범위) | 자동 | params 저장·실행 반영 |
| 프롬프트 향상기 | 자동 | 의도 → 시스템프롬프트 생성 |
| 워크플로 저장/발행/실행/실행로그 | 자동 | 저장 → 실행 → node_runs 기록 |
| RAG 컬렉션 생성·권한(allowed_roles) | 자동 | 생성, role별 가시성 |
| 지식 업로드(txt/pdf/docx) | 자동 | 추출·인덱싱, 손상 파일 400 |
| 사용자 목록·role 변경 | 자동 | role 변경 + 토큰 무효화 |
| 통계 / 피드백 검수 | 자동 | stats 집계, 유저 피드백 열람 |
| 에이전트/워크플로 빌더 UI | 수동 | 모달·노드 편집·발행 버튼 |

### 4.3 유저 (Gateway user)
| 기능 | 방식 | 합격 기준 |
|---|---|---|
| 발행 에이전트 목록·사용 | 자동 | published만 보임, 실행 200 |
| chat + 대화 이력/요약/변수폼 | 자동 | 멀티턴 맥락 유지, 변수 주입 |
| chat+RAG (권한 컬렉션만) | 자동 | 허용 컬렉션 답변, admin-전용 제외 |
| 딥리서치 HITL | 조건부(검색키)/수동 | 계획 검토 → 승인 → 보고서 |
| slides / sheets | 자동 | 산출물 생성 |
| image / page / podcast | 자동(현 키) | png/html/mp3 생성 |
| 피드백 👍/👎 | 자동 | 저장 → admin 큐 노출 |
| RBAC 차단 | 자동 | admin·Control Plane 접근 403 |
| 채팅/이력/변수폼/피드백 UI, 미디어 산출물 열람 | 수동 | UX·시각 확인 |

## 5. 산출물

1. **자동화 스크립트** `scripts/role_e2e.py`
   - Part 1 E2E 줄기 + Part 2의 모든 [자동] 항목.
   - 역할별 토큰으로 실행, 각 단계 PASS/FAIL/SKIP 출력, 말미 집계.
   - Control Plane `:8090`와 Gateway `:8080`가 떠 있어야 함(미기동 시 CP 항목 SKIP).
   - 기존 `scripts/e2e_smoke.py`와 별개(그건 단일 게이트웨이 스모크). 역할/권한 관점 전용.
2. **수동 UAT 체크리스트** `docs/superpowers/uat-checklist.md`
   - [수동] 항목을 역할별 체크박스 + 기대결과로 정리. 출하 전 1회 사람이 훑는 용도.

## 6. 합격 기준 (출하 게이트)

- 자동: 모든 [자동] 항목 PASS. SKIP은 사유(미구성)와 함께 허용.
- 수동: 체크리스트 전 항목 ✅.
- **권한 경계 항목(403 / 컬렉션 격리 / 토큰 무효화 / 라이선스 회수 / hot-reload)은
  반드시 PASS** — 하나라도 실패하면 출하 차단.

## 7. 범위 밖 (YAGNI)

- 부하/성능 테스트(별도 작업), 멀티 인스턴스 HA(현 단일 서버 결정), 실 Google OAuth
  흐름(실 클라이언트 ID 필요 — 수동 1건만 선택적 확인).
